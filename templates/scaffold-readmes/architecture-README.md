# Architecture (WARM tier)

System design, data flows, component contracts. Loaded on-demand, not at session start.

- One file per subsystem: `[system]-architecture.md` (~500 lines max)
- Document the "why" and the boundaries, not the line-by-line code
- Update when interfaces or invariants change; otherwise leave alone
- Do NOT put: active task notes (HOT), code snippets (patterns/), historical decisions (backlog/archive/)

See `../TIERED_DOCUMENTATION_GUIDE.md`.
