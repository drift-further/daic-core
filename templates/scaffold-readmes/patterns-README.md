# Patterns (WARM tier)

Reusable code patterns with Do/Don't examples. Loaded on-demand.

- One file per pattern family: e.g. `error-handling.md`, `async-patterns.md` (~400 lines max)
- Each pattern: problem, solution, Do example, Don't example, when-not-to-use
- If it appears three times in the codebase, it belongs here
- Do NOT put: one-off code, architecture narrative (architecture/), task logs

See `../TIERED_DOCUMENTATION_GUIDE.md`.
